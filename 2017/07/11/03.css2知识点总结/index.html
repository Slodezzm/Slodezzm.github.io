选择器 —————

  1. 基本选择器，

    1. 标签选择器，所有的标签都能做标签选择器，且能够找到页面上所有的该类标签
      p {color:red} 页面上所有的p标签字体颜色都会成红色
    2. 类选择器，能够找到页面上属于该class的标签
      每一个类要尽可能小，有“公共”的概念，能够让更多的标签使用。
      每个标签都可以属于多个类，一个类可以作用于任何的标签
      .className {}; .one {color:red}; 页面上所有类名为one的标签字体颜色都是红色。
      类名中只能包括字母，数字，下划线，中连接线，且不能以数字开头，且不能以中连接线+数字开头

    3. id选择器，因为id的唯一性，所以能够找到唯一对应的id，id名不能和标签名相同
      id名中只能包括字母，数字，下划线，中连接线，且不能以数字开头，且不能以中连接线+数字开头；最好是有意义
      ————– 类和id 的区别 —————————
      类相当于 人名 可以给多个标签设置相同的类名， 我们可以给一个标签设置多个类名 class= “one two”；
      id相当于是人的身份证一样 一个标签最好是一个id名， 一个id名只能设置一个标签，id其实不是给css用的， 是给js用的

    4. 后代选择器， 用空格隔开，比如.div p表示的是所有包裹在div种的后代p，

      只要是嵌套在这个标签里面的标签，都是这标签的后代
    5. 交集选择器，比如div.aab，表示既是div又是类.aab的那个div标签，注意中间不能有空格，
      有标签选择器，和类选择器一起构成交集   ，标签选择器要写在前面
    6. 并集选择器，用逗号表示，例如div，p表示页面上所有的div和p都被选中
    7. 通配符*，表示选中了页面上所有的元素（标签）；效率非常低，实际开发中不推荐使用。
      元素的查找顺序，从右往左找

      元素权重的比较原则：
      选择器从右往左开始判断是否被选中

      判断该元素是否被选中，如果被选中了，数权重，（权重的计算方法：id的数量，类的数量，标签的数量）。如果权重一样，则在样式表里谁写在后面听谁的。
判断该元素是否被选中了，如果没有被选中，比较谁描述的最近（就近原则）；如果描述的是同样的近，则开始比较权重；如果权重再一样，则在样式表里谁写在后面听谁的。
可以提升被选中的任何元素的某一个属性的权重为无穷大：！important，格式为k：v ！important；
但是如果某个元素未被选中，样式通过继承的，则！important不能提升该元素任何属性的权重。

      关于padding和margin几个实用的冷门技巧：
      1. 当使用%百分比表示padding任意方向的大小时，都是以盒子父亲的宽度为基准；---可以理解为只有父盒子的宽度可以继承，而父盒子的高度不可以继承。
2. 当使用%百分比表示margin任意方向的大小时，都是以盒子父亲的宽度width为基准
3. 当使用em表示padding任意方向的大小时，都是以盒子中内容字体大小为基准；---看到em就和字体或者行高有关
4. 当使用em表示margin任意方向的大小时，都是以盒子中内容字体大小为基准；---看到em就和字体或者行高有关
    2. 盒子内边距：padding，padding-top、right、bottom、left
      计算盒子的大小
      盒子真实占用宽 =  border-left-width  + padding-left +width  + pdding-right  +border-right-width
盒子真实占有高 = border-top-width + padding-top  +height + padding-bottom + border-bottom-width
页面上布局，如果给一个盒子留的位置是固定的，如果你给这个盒子加了边框或者padding，那么你就要相应的减小内容。 总之就是要保证盒子的大小不变，从而不影响布局，不影响其他的元素 。
      padding特殊：
      一般的情况下，块级元素只设置padding会改变盒子的大小 ----小盒子的宽是由大盒子继承过来的，设置小盒子的左右padding，不会改变小盒子的大小。

  5. 脱标的常用三种方式：脱标之后没有行内和块级元素的等级了，

    1. float:left; float:right; 浮动的特点；

      1. 浮动的元素脱离标准流，没有行内和块级之分
      2. 浮动的元素互相贴靠，前提：在同一个结构下;比如上下两个浮动的元素中间隔着标准流，浮动不会越过标准流，去找另一个浮动的元素 —-> 浮动的元素找浮动的元素，不浮动就找不浮动
      3. 浮动的元素有字围效果，标准流中的 文字不会被浮动起来的 元素遮挡住；
      4. 收缩:一个浮动的元素，如果没有设置width，那么将自动收缩为文字的宽度;特点和行内块一样;可以设置宽高,同时一行上可以显示多个
      5. 清除浮动的方法 核心:clear:both —-> 两个浮动的元素上下并列时,后面的元素会去贴靠前面的元素，有时候不需要这个特性
        1. 给处在标准流中的父元素设置高度：给父盒子加高度，缺点是不能适应页面的快速变化
        2. 添加属性:clear:both, 表示自己内部的元素不受其他盒子的影响，缺点是margin失效了
        3. 额外标签法，给这个额外的标签设置clear：both ;相当于在两部分浮动元素中间建一个墙，隔开两部分浮动，让后面的浮动元素，不去追前面的浮动元素。
          内部标签法：父亲有高度了 ----内墙法:给前面的浮动元素建立内墙
外部标签法，父亲还是没有高度 ---在前后两个浮动的元素中间建墙
        4. overflow:hedden; 在父元素中使用overflow:hidden来清除浮动。弊端：如果使用负值定位，会被隐藏 /这个现象，不能解释，就是浏览器的小偏方/

定位

  1. 定位的属性值：x方向，left表示从左往右多少像素，right，表示从右往左有多少像素; y方向：top表示从上往下有多少像素；bottom表示从下往上有多少像素;当给定像素是负数时候，移动方向和正值相反

  2. 定位的类型:

    1. static 静态定位，盒子默认static,元素不脱标

    2. realatIve，相对定位 —-特点:不脱标，老家留坑，形影分离
      格式:position:realatIve;left100px:top200px;
      作用:1.微调元素，2.作为绝对定位的参考，子绝父相

    3. absolute，绝对定位, —-绝对定位特点：脱离标准流
      绝对定位的元素，直接从父元素 border 作为定位值的起始参考
      如果小盒子在大盒子中要定位并且水平居中：left：50%；margin-left：负的宽度（width）的一半，同样垂直居中同理
      绝对定位的元素参考：
      1.如果元素没有父元素，2.如果绝对定位的元素有父元素，但是父元素没有定位，都”找干爹“
      参考点：如果用top描述，定位参考点是页面的左上角，不是浏览器的左上角
如果是bottom描述，定位参考点是当前html文档的首屏窗口对应的左下角
      1. 如果绝对定位的元素有父元素，而且父元素有定位（非static），那么这个绝对定位的元素偏移是以自己的父元素为基础
        一个绝对定位的元素，如果父辈元素中出现了定位了（可以是相对，绝对，固定）的元素，那么将以父辈这个元素，为参考，以最近的父辈元素为准

    4. fixed，固定定位，固定定位脱离标准流
      固定定位，相对浏览器窗口定位，无论页面如何滚动，盒子显示的位置不变

      trbl永远是相对于浏览器的边框来的

      5.和定位元素相关的属性:z-index，表示层级，只能设置在非static定位元素上，
      1.只有定位了的元素才有z-index
      2.如果大家都没有z-index值，或者z-index值一样，那么谁写在html后面，谁在上面能压住别人，定位了的元素永远能够压住没有定位的元素
      3.从父现象:父元素的z-index值小，儿子(子元素)再牛逼都没有用

    css精灵技术

  6. “css精灵”，英语css sprite，所以也叫做“css雪碧”技术。是一种CSS图像合并技术，该方法是将小图标和背景图像合并到一张图片上，然后利用css的背景定位来显示需要显示的图片部分。
  7. css精灵有什么优点，就是减少了http请求。比如4张小图片，原本需要4个http请求。但是用了css精灵，小图片变为了一张图，http请求只有1个了。
  8. 精灵图的使用:
    1. 盒子大小一定要和需要显示的小背景图大小一样
    2. background-position：-x -y
    3. 制作精灵图，切片导出，用魔法棒把背景去掉，再倒入新的图片中

零散的—-知识点：

  1. 定位元素如果同时设置在x方向left和right那么以left为准，right不起作用，在y方向上同时设置top，bottom时以top为准
  2. 盒子属性里、文本属性等属性里能设置负值的是margin，和z-index，text-indent
  3. 给父盒子设置overflow：hidden属性后，原先溢出后横跨在父盒子border上的内容也会被隐藏，也就是只能显示父盒子border内的内容
  4. 背景应用于由内容、padding、边框组成的区域，边框也可以被背景渲染。
  5. css样式初始化
    1. 清除标签的margin和padding:html,body,ul,li,ol,dl,dd,dt,p,h1,h2,h3,h4,h5,h6,form,fieldset,legend,img{margin:0;padding:0}
    2. 清除img的边框img { border: 0};
    3. 清除li标签之前的小点ul{list-style: none;}
    4. 给页面设置一个统一的字体大小和颜色，还要设置body的背景颜色。body {font-size: 12px;color: #434343;font-family: “宋体”;background-color:“#ffffff”}
    5. 清除浮动
      .clearfix:after {content: “”;height: 0;line-height: 0;display: block;visibility: hidden;clear: both;};.clearfix {_zoom: 1;//兼容ie6}

      a {color: #434343;text-decoration: none;}
      a:hover {color: green;}
    7. 设置浮动：
      .fl {float: left;}

  7. 浮动和绝对定位和固定定位的盒子，脱离标准流都不会再继承父亲的宽度,需要手动设置宽高;
  8. 元素的嵌套
    1. a标签可以嵌套所有的元素 —display:block;
